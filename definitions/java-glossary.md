# 운영체제

*OS, Operating System*

컴퓨터 하드웨어를 관리하고, 다른 프로그램이 CPU·메모리·저장장치·입출력 장치 등을 사용할 수 있도록 중간에서 연결·조율하는 **시스템 소프트웨어**입니다. Windows, macOS, Linux 등이 대표적입니다.

---

# 메모리

*Memory, RAM, 주기억장치*

프로그램 코드와 실행 중인 데이터가 올라가 **CPU가 읽고 쓸 수 있는 저장 공간**입니다.

---

# 오픈소스

*Open Source, 오픈 소스*

**소스 코드가 공개**되어 누구나 열람·수정·재배포할 수 있도록 허용하는 소프트웨어·라이선스 모델을 말합니다.

---

# 클래스 로더

*Class Loader, 클래스로더*

JVM이 실행에 필요한 **`.class` 바이트코드를 찾아 읽고**, 메서드 영역 등 **메모리에 적재(로드)**하는 JVM 하위 시스템입니다.

---

# 인터프리터

*Interpreter, 해석기*

바이트코드 명령을 **한 단위씩 순서대로 읽어 곧바로 실행**하는 방식입니다.

---

# JIT 컴파일러

*JIT, Just-In-Time Compiler, 제때 컴파일러*

실행 중에 **자주 호출되는 코드(핫스팟)**를 감지해 바이트코드를 **네이티브 기계어로 번역·캐시**하는 최적화 구성 요소입니다.

---

# 가비지 컬렉터

*GC, Garbage Collector, 가비지 컬렉션*

더 이상 도달할 수 없는(참조되지 않는) 객체가 차지한 **힙 메모리를 자동으로 회수**하는 JVM 메커니즘입니다. 개발자가 직접 해제하지 않아도 되지만, **수집 시점·일시 정지(stop-the-world)** 등 실행 특성을 이해하는 것이 중요합니다.

---

# JVM

*Java Virtual Machine, 자바 가상 머신*

`.class` **바이트코드를 읽어 실행**하는 가상 기계(런타임)입니다. 운영체제 위에서 동작하며, 클래스 로더·실행 엔진·가비지 컬렉터 등으로 구성되어 **한 번 작성한 자바 프로그램을 여러 플랫폼에서 비슷하게** 돌릴 수 있게 합니다.

---

# JDK

*Java Development Kit, 자바 개발 키트*

자바 **프로그램을 개발·빌드**할 때 쓰는 도구 묶음입니다. 컴파일러(`javac`), 디버거, 라이브러리, 문서 도구 등이 포함되며, **실행용 JRE(또는 이에 상응하는 런타임 구성)**를 함께 제공하는 배포 형태가 일반적입니다.

---

# JRE

*Java Runtime Environment, 자바 실행 환경*

자바 **애플리케이션을 실행**하는 데 필요한 환경입니다. JVM과 핵심 클래스 라이브러리 등으로 이루어지며, **컴파일러는 포함하지 않는** 구성이 전형적입니다(개발은 JDK가 담당).

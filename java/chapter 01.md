# Chapter 01 - 자바 소개와 개발 환경 준비

## 1) 🎯 학습 목표

- 자바 언어의 특징과 장점을 설명할 수 있다.
- JDK, JRE, JVM의 역할 차이를 구분할 수 있다.
- 자바 개발 환경을 구성하고 첫 프로그램을 실행할 수 있다.

## 2) 📚 학습 내용

### 2-1. 핵심 개념

- 자바(Java): 객체지향 프로그래밍을 지원하는 대표적인 범용 언어
- JVM(Java Virtual Machine): 운영체제와 무관하게 자바 프로그램을 실행해주는 가상 머신
- JDK/JRE 차이:
  - JDK는 개발 도구(컴파일러 포함)까지 제공
  - JRE는 실행 환경만 제공

### 2-2. 문법/코드 예제

```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, Java!");
    }
}
```

- 코드 설명:
  - 1) `public class Main`은 자바 클래스의 시작을 의미한다.
  - 2) `main` 메서드는 프로그램 실행 시작 지점이다.
  - 3) `System.out.println()`은 콘솔에 문자열을 출력한다.

### 2-3. 자주 하는 실수

- 파일명(`Main.java`)과 클래스명(`Main`)을 다르게 작성함
- `main` 메서드 시그니처를 잘못 입력함
- JDK 설치 후 환경 변수(PATH) 설정을 누락함

### 2-4. 확인 문제

1. JVM이 필요한 이유를 한 문장으로 설명해보자.
2. JDK와 JRE의 차이는 무엇인가?
3. `main` 메서드가 하는 역할은 무엇인가?

## 3) 🧾 요약 정리

- 자바는 JVM 기반으로 동작해 높은 이식성을 가진다.
- 개발에는 JDK가 필요하고, 실행에는 JRE/JVM이 필요하다.
- 첫 프로그램 실행을 통해 자바 개발 흐름(작성-컴파일-실행)을 익혔다.

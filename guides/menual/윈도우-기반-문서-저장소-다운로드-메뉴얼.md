# Git/Obsidian 설치 및 저장소 다운로드

[README로 돌아가기](../../README.md)

이 문서는 Windows 환경에서 **powershell 터미널을 사용해** `Git`과 `Obsidian`을 설치하고,  
이 프로젝트를 Obsidian Vault로 여는 방법을 간단히 안내합니다.

## 1. Git 버전 확인 후 설치 (터미널)

먼저 설치 여부를 확인합니다.

```powershell
git --version
```

버전이 정상 출력되면 설치를 건너뜁니다.  
버전이 출력되지 않거나 `git` 명령을 찾지 못하면 아래 명령으로 설치합니다.

```powershell
winget install --id Git.Git -e --source winget
```

설치 후 다시 확인:

```powershell
git --version
```

## 2. Obsidian 설치 여부/버전 확인 후 설치 (터미널)

먼저 설치 여부와 버전을 확인합니다.

```powershell
winget list --id Obsidian.Obsidian
```

목록에 Obsidian과 버전이 보이면 설치를 건너뜁니다.  
표시되지 않으면 아래 명령으로 설치합니다.

```powershell
winget install --id Obsidian.Obsidian -e --source winget
```

설치 후 다시 확인:

```powershell
winget list --id Obsidian.Obsidian
```

## 3. 저장소 클론

원하는 작업 폴더로 이동 후, 저장소를 클론합니다.

```powershell
git clone https://github.com/Jun983/java-document.git
```

이미 클론된 상태라면 이 단계는 건너뜁니다.

## 4. Obsidian에서 프로젝트 열기

1. 윈도우 시작 메뉴에서 `Obsidian` 검색 후 실행
2. **Open folder as vault** 선택
3. Git으로 clone한 경로에서 `java-document` 폴더 선택
4. Vault 열기

## 5. 커뮤니티 플러그인 활성화

이 저장소에는 용어사전 등을 위한 **커뮤니티 플러그인**(`Note Definitions`) 파일이 이미 포함되어 있습니다.  
다만 Obsidian에서 **커뮤니티 플러그인 사용이 꺼져 있으면** 해당 기능이 동작하지 않을 수 있으므로, Vault를 연 뒤 아래를 확인합니다.

1. 왼쪽 하단 **톱니바퀴(설정)** 을 엽니다.
2. 왼쪽 목록에서 **커뮤니티 플러그인**을 선택합니다.
3. **제한 모드**가 켜져 있으면 끕니다. (끄면 커뮤니티 플러그인을 사용할 수 있습니다.)
4. 필요 시 **안전하지 않은 커뮤니티 플러그인 사용 허용** 등 안내에 따라 신뢰·허용을 진행합니다.
5. **설치된 플러그인** 목록에서 **Note Definitions**가 **켜짐** 상태인지 확인합니다. (저장소에 포함된 플러그인이므로 별도로 스토어에서 받을 필요는 없습니다.)

이후 노트에서 용어에 마우스를 올리거나 읽기 보기에서 정의가 보이는지 확인해 보면 됩니다.
